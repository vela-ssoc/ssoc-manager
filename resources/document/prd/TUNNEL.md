# TUNNEL

> 节点间的长连接通信是整个 ssoc 通信网络的基础设施，在 v1 版本中，为了满足多路复用需求，自己开发了一套 TCP 多路复用库。

## 问题

- **维护复杂**

多路复用库属于底层通信基础设施，代码复杂，长期由个人维护。

- **生态匮乏**

自研实现缺乏社区验证，所有的运行表现仅在 ssoc 这个小圈子，缺乏极端的运行场景。

- **非业务核心**

多路复用协议虽为 ssoc 的通信核心，但非业务的核心，该类基础设施持续投入产出比不合理。

## 决策

在 v2 版本中，明确放弃自研 TCP 多路复用实现，转而采用成熟的开源方案，依靠社区的力量。

## 实现

- `github.com/xtaci/smux`

- `github.com/hashicorp/yamux`

v2 版本将采用上面的两个库作为通信多路复用库。两者均为热门使用的 TCP 流多路复用库，社区较为活跃，截止到当前（2026-02-05）：
`smux` 为 1.5k stars，`yamux` 为 2.5k stars，其中 `yamux` 还是知名的 [HashiCorp](https://www.hashicorp.com) 公司项目。

v2 通信层同时引入 `smux` 与 `yamux`，二者为互备和互相验证关系，当其中一方出现 BUG 而不能及时升级时，可以切换到另一个通信库。

## 亮点

> v2 将新增的功能。

### 限速

v2 tunnel 将原生集成流量限速功能，该能可以动态调节，控制 tunnel 的发送/接收速率。

在一些意外场景下，比如：agent 上的业务流量过大，导致 agent 程序抢占宿主机的网络资源，可以对该 agent 的 tunnel 进行限速。

### 统计

tunnel 将会统计虚拟子流的连接数，传输/发送字节数。结合 VictoriaMetics 监控 tunnel 的负载状况。
